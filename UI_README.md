# Actuarial Stochastic Model — Offline Result UI (`ui_app.html`)

Zero-install, fully offline viewer for the completed stochastic capital model. It
**consumes only model output** — it never re-runs a calculation. Open it and read.

## How to use it
1. Double-click **`ui_app.html`** (any modern browser: Chrome, Edge, Firefox, Safari).
2. No Python, Node, server, internet, or install is required. The data snapshot is
   embedded inline, so the page works from a USB stick or an air-gapped machine.
3. (Optional) Drag a different `ui_data.json` onto the page to load another snapshot.

## What you can see — core tabs + phase deep-dives

In addition to the five core tabs below, phase-specific deep-dive tabs surface the
sophistication track: Joint Actions (P24), Path-wise Actions (P25), Full Re-Agg
(P26), and **Skew-t Tail (P27)** — the latter showing skew-t vs symmetric-t vs
nested SCR, the upper/lower tail-dependence asymmetry profile (95% CI), the
gamma_hat~0 material finding, the bootstrap CI, and MR-015.
The **Grouped-t Tail (P28)** tab adds grouped-t vs single-df t vs nested SCR,
the grouped-t bootstrap CI, within/cross upper/lower tail-dependence and
cross-block dilution, residual widening, and MR-016.

### Five core tabs
- **Overview** — headline roll-up (tasks/phases/gates/risks, nested 99.5% SCR) and verdicts.
- **Inventory & Contract** — every model-output artifact, SHA-256 content-addressed, with a
  name/category filter and the `ui_data.json` contract schema.
- **Calibrations** — per-driver explorer (GBM, HW1F, G2++ 2F, CIR++, OU-lapse, mortality-trend): gate
  criteria, calibrated parameters, and inline-SVG fit-diagnostic charts.
- **Capital & Tail** — G2++ five-driver SCRs, standalone→var-covar→copula→nested aggregation, VaR/ES with
  95 % bootstrap CIs, and convergence vs outer-count with the recommended n* marker.
- **Governance** — deployment gates, the model-risk register (filterable impact×likelihood
  heatmap), the ChangeRecord approval trail with sign-off chains, and a recomputed audit-integrity badge.

## Packaging & polish (UI Task 5)
- **Export charts (PNG)** — serialises the visible inline-SVG charts on the active tab to PNG
  (2× scale, themed background) via an in-browser canvas. No network, no library.
- **CSV exports** — one click each for the **Inventory**, **Risk register**, and **Change records**
  as RFC-4180-style CSV (quoted, newline-stripped cells) that open cleanly in Excel.
- **Print / Save PDF** — a dedicated print stylesheet expands every tab onto the page with a light
  theme, per-section titles, and chart/table page-break protection; use the browser's
  "Save as PDF" to archive a full report.
- **Accessibility** — tabs and sub-navs expose `tablist`/`tab` roles with `aria-selected`, roving
  `tabindex`, arrow-key/Home/End navigation and Enter/Space activation; filters carry `aria-label`s;
  a visible focus outline is provided throughout.

## Offline guarantee (verified each build)
`scripts/ui_app_self_test.cjs` loads the page under jsdom, blocks `fetch`/`XHR`, clicks every tab
and sub-view, exercises the CSV builders and ARIA roles, and asserts **0 network calls / 0 JS
errors**. An external-reference scan confirms no `http(s)://`, CDN, `<script src>`, `<link>`, or
`@import` in the shipped file (the sole `xmlns` is added at runtime only when exporting a chart).

## How it is rebuilt (maintainers)
`ui_app.html` is generated by `scripts/build_ui_data.py`, a **no-calculation bundler** that scans
the model's already-produced output JSONs, normalises them into the stable `ui_data.json` contract
(v1.10.0), and embeds that snapshot inline in an HTML template:

```
PYTHONPATH=. python3 scripts/build_ui_data.py        # writes ui_data.json + ui_app.html
node scripts/ui_app_self_test.cjs ui_app.html         # must print "ok": true
```
